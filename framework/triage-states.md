# Triage States

A glossary of the triage states used in this repo's issue tracker. Use it to map maintainer phrasing about issue state to canonical state values — when the maintainer asks for a state change in vague terms ("mark this ready", "park this", "shipped"), pick the row whose meaning fits.

| State             | Meaning                                             |
| ----------------- | --------------------------------------------------- |
| `needs-triage`    | Maintainer needs to evaluate this issue             |
| `needs-info`      | Waiting for more information                        |
| `ready-for-agent` | Fully specified, ready for an agent                 |
| `ready-for-human` | Requires a human to pick up — an agent cannot start |
| `in-review`       | Shipped by implementer, awaiting maintainer signoff |
| `wontfix`         | Will not be actioned                                |
| `done`            | Verified and accepted by maintainer                 |
