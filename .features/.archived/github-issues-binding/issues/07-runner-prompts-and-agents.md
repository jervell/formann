## What to build

Rewrite the runner's review-and-gate prompt and refresh the surrounding narrative so nothing presumes tracker writes are git commits or that the binding-native ref shape is `<feature>/NN`. Three pieces land together:

1. **`framework/runner/review-and-gate.md` step 6 rewrite.** Replace "Commit as a single `tracker:` commit" with binding-agnostic verb-shaped instructions deferring to BINDING.md ("set the state to `<verdict>`, comment with `Review (AFK gate)`"). 2. **`review-issue` agent commit-scanning regex update.** Today the agent finds issue-completing commits by scanning messages on the current branch for `<feature>/NN` patterns. Update the scanning to recognize **both** `#N` (GH binding) and `<slug>/NN` (local-markdown). The agent already knows the feature from its branch context.

3. **Bail-flow narrative refresh.** In `framework/afk-runner.md` and `framework/lifecycle.md`, clarify that `tracker:`-prefixed commits and the commit-then-propagate dance are local-markdown's mechanics, not a runner contract. Under GH binding, a `/implement` bail makes an API call and lands no commit; the abort-flag logic must still work in the no-commit case. Also refresh the comments around `gate_dirty → gate-failed` in `run-the-queue.sh` to drop the file-binding-specific rationale; the check itself stays universal.
