# Strengthen `list-archived-excluded` to actually demonstrate exclusion

> *This was generated by AI during to-issues.*

## Parent

`.features/github-issues-binding/PRD.md`

## What to build

The fixture at `framework/bindings/issue-tracker/github-issues/tests/fixtures/list-archived-excluded/recorded-responses/graphql-response.json` contains only OPEN parents (`#3 alpha-feature`, `#7 beta-feature`). The test `--list: archived (closed) parent absent — only open parents returned` therefore proves only that two open features survive — the trivial open-only baseline. The promised exclusion behaviour relies entirely on the GraphQL `states: [OPEN]` clause and is not exercised by any test.

Pick one of:

- **(a) Strengthen the script-level filter.** Augment the fixture to include a CLOSED `formann:feature`-labelled parent in the recorded response. The `gh` shim serves the response verbatim regardless of the query, so to make the test meaningful the `tracker-snapshot` script must explicitly drop closed entries (or assert at the script level that closed entries don't make it into the output). Update `expected.json` to assert the closed parent is filtered out.
- **(b) Assert the query string directly.** Have the `gh` shim record the full query string, and add a separate test that grep's the recorded query for `states: [OPEN]`. Pair this with a fixture covering deduplication only, and rename the existing test so its name matches what it verifies.

Either way: no test in `tracker-snapshot.bats` should claim to verify archived-parent exclusion without actually exercising it, and the `states: [OPEN]` contract must have at least one direct check.
