# Bootstrap Formann's self-install

> *This was generated by AI during to-issues.*

## Parent

`.features/self-install/PRD.md`

## What to build

One-off live operation that lands the dogfood loop. Run the self-install-aware `installer/install.sh .` from the Formann root with `FORMANN_INSTALL_BINDING_issue_tracker=local-markdown` set so the role-binding prompt is bypassed, then `git add .gitignore && git commit` with a `tracker:`-prefixed message. After the bootstrap, Formann's `.gitignore` carries the managed block, the per-machine symlinks (`.claude/skills/*`, `.claude/agents/*`, `docs/formann/*`) and `runner/Dockerfile` exist locally but are gitignored, `git status` is clean, and the maintainer can invoke `/triage`, `/implement`, and the AFK runner from inside the Formann checkout.

## Acceptance criteria

- [ ] `installer/install.sh .` runs from the Formann root without interactive prompts (binding env var set)
- [ ] Post-install `.gitignore` contains the managed block with the marker pair and every expected product listed
- [ ] Post-commit `git status` reports a clean working tree — every installer product is either committed (the `.gitignore` change itself) or gitignored
- [ ] `.claude/skills/*` symlinks exist locally and resolve through `.formann` to the corresponding `framework/skills/*` content
- [ ] `.claude/agents/*` symlinks exist locally and resolve similarly
- [ ] `docs/formann/issue-tracker` symlink resolves to `framework/bindings/issue-tracker/local-markdown`
- [ ] `docs/formann/<doc>.md` flat role docs resolve to `framework/<doc>.md`
- [ ] `runner/Dockerfile` exists as a real file (scaffolded from the installer template) and is gitignored
- [ ] Commit message starts with `tracker:` per the binding's commit convention
- [ ] Invoking a framework skill (e.g. `/triage`) from inside the Formann checkout works — Claude Code discovers it via `.claude/skills/triage` [human]

## Blocked by

- `self-install/01` — the installer must support self-install mode before bootstrap is meaningful.
- `self-install/04` — the runner must see framework wiring from host `.claude/` before the bootstrap's [human] AC ("Claude Code discovers `/triage` via `.claude/skills/triage`") can be verified end-to-end on a self-installed Formann.

## Agent Brief

> *This was generated by AI during triage.*

**Summary:** Run the self-install-aware installer on Formann itself, commit the resulting `.gitignore` change with a `tracker:`-prefixed message, and verify the dogfood loop works end-to-end.

**Current behavior:**
Formann's repo has no `.claude/skills/*`, `.claude/agents/*`, `docs/formann/*` symlinks and no `runner/Dockerfile`. Claude Code working inside the Formann checkout cannot discover the framework's skills or agents — only the user's global skills and any plugins are visible. The AFK runner cannot run from inside Formann because there is no `runner/Dockerfile` to build. `.gitignore` contains only the pre-existing global ignores (`.DS_Store`, `/.idea/`, `*.iml`, `/.claude/settings.local.json`, `__pycache__/`).

**Desired behavior:**
- From the Formann repo root, run:
  ```
  FORMANN_INSTALL_BINDING_issue_tracker=local-markdown ./installer/install.sh .
  ```
- The installer (post-#01) detects self-install via canonical-path equivalence, creates `.formann -> framework/`, creates the per-component symlinks under `.claude/skills/`, `.claude/agents/`, `docs/formann/`, scaffolds `runner/Dockerfile` from the installer's template, and writes the managed gitignore block to `.gitignore`.
- Stage `.gitignore` and commit with a `tracker:`-prefixed message describing the bootstrap.
- Post-commit, `git status` is clean: every installer product is either gitignored (everything except `.gitignore` itself) or committed (the `.gitignore` change).
- The bootstrap lands on the current `self-install` branch (not `main`) — this branch ships the installer change together with the bootstrap commit and merges to `main` as one coherent feature.

**Key interfaces:**
- **Installer CLI:** `installer/install.sh <consumer-path>` invoked with the Formann root as the consumer-path argument (e.g. `./installer/install.sh .` or `installer/install.sh /path/to/formann`). Either form works; the installer canonicalizes the path before the self-install check.
- **`FORMANN_INSTALL_BINDING_issue_tracker` env var:** set to `local-markdown` to bypass the role-binding prompt. Per the installer's existing convention (`FORMANN_INSTALL_BINDING_<role>` with dashes replaced by underscores).
- **The single tracked artifact:** `.gitignore`. Every other installer product is either gitignored (`.formann`, `.claude/*` symlinks, `docs/formann/*` symlinks, `runner/Dockerfile`) by the managed block written in self-install mode, or pre-existing.

**Acceptance criteria:**
- [ ] `installer/install.sh .` (or equivalent invocation with absolute path) runs from the Formann repo root and completes successfully without interactive prompts
- [ ] Post-install `.gitignore` contains the managed block with the marker pair and every expected installer product listed (per `self-install/01`'s output spec)
- [ ] `.formann` symlink exists at the Formann root and resolves to `framework/`
- [ ] `.claude/skills/<name>` symlink exists for every entry under `framework/skills/` (currently: `grill-me`, `grill-with-docs`, `implement`, `tdd`, `to-issues`, `to-prd`, `triage`), each resolving via `.formann` to `framework/skills/<name>`
- [ ] `.claude/agents/<name>.md` symlinks exist for every entry under `framework/agents/` (currently: `review-feature.md`, `review-issue.md`)
- [ ] `docs/formann/issue-tracker` symlink resolves to `framework/bindings/issue-tracker/local-markdown`
- [ ] `docs/formann/<doc>.md` flat role docs exist and resolve to `framework/<doc>.md` for every flat doc the installer manages (`lifecycle.md`, `inbox.md`, `domain.md`, `triage-states.md`, `afk-runner.md`, `afk-runner-flow.md`)
- [ ] `runner/Dockerfile` exists as a real file (scaffolded from `installer/templates/Dockerfile`) and is matched by the managed gitignore block
- [ ] A `tracker:`-prefixed commit lands on the `self-install` branch staging `.gitignore` (and only `.gitignore`)
- [ ] Post-commit `git status` reports a clean working tree
- [ ] Invoking `/triage` (or another framework skill) from inside the Formann checkout works — Claude Code discovers the skill via `.claude/skills/triage` [human]

**Out of scope:**
- Editing `runner/Dockerfile` post-scaffold — the bootstrap leaves the installer template untouched; consumers (including Formann) edit it later if they need to
- Pasting the installer's printed CLAUDE.md snippet into Formann's `CLAUDE.md` — CLAUDE.md retains its current contents per the PRD
- Merging `self-install` to `main` — the merge is a separate maintainer step after all three issues are done and the feature is archived
- Re-running the bootstrap on subsequent machines — each contributor runs the installer once per machine (per ADR-0001 — `.formann` is per-machine)
- Adding tests for the bootstrap — the bootstrap is verified by post-condition (clean `git status`, expected symlink graph), not by automated tests; `self-install/01` already covers the installer's behavior in tests
