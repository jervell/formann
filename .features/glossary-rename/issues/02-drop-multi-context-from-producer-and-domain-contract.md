# Drop multi-context from the producer-side format spec, skill prompt, and domain-doc contract

> *This was generated by AI during to-issues.*

## Parent

`.features/glossary-rename/PRD.md`

## What to build

Collapse the producer-side and the consumer-side domain-doc contract to one path — a single `GLOSSARY.md` at the consumer's repo root, with no escape hatch back to multi-context. This slice is the deliberate scope reduction half of the rename, separate from the mechanical reference sweep.

Three files change:

- `framework/skills/grill-with-docs/CONTEXT-FORMAT.md` is renamed to `framework/skills/grill-with-docs/GLOSSARY-FORMAT.md` via `git mv`. Inside the spec, the inner H2 in the example becomes `## Terms` and the "Single vs multi-context repos" section (including the `CONTEXT-MAP.md` example and the skill-inference list) is deleted entirely. Sibling naming with `ADR-FORMAT.md` is preserved.
- `framework/skills/grill-with-docs/SKILL.md` drops its multi-context block. The "File structure" subsection collapses to a one-line prose mention. The internal link to `CONTEXT-FORMAT.md` repoints at `GLOSSARY-FORMAT.md`. Remaining `CONTEXT.md` mentions in the prompt become `GLOSSARY.md`.
- `framework/domain.md` drops the "This is a single-context repo:" framing and its ASCII tree. The opening list points at `GLOSSARY.md` instead of `CONTEXT.md`, and file structure is described in one line of prose without a diagram.

**Summary:** `git mv` the producer-side format spec to `GLOSSARY-FORMAT.md`, delete every multi-context branch from the format spec, the `grill-with-docs` skill prompt, and the consumer-side domain-doc contract, so the docs describe exactly one path: a single `GLOSSARY.md` at the consumer's repo root.

**Current behavior:**
The producer-side format spec is named `CONTEXT-FORMAT.md` and contains a "Single vs multi-context repos" section with a `CONTEXT-MAP.md` example and a skill-inference list (which-shape-applies rules). The `grill-with-docs` skill prompt carries a parallel single-vs-multi-context branch in its "File structure" section, including a second ASCII tree for the multi-context layout, and refers to `CONTEXT.md` and `CONTEXT-FORMAT.md` by name. The consumer-side domain-doc contract opens with "This is a single-context repo:" followed by an ASCII tree, and its bullets and prose reference `CONTEXT.md`. The skill prompt's YAML frontmatter `description:` field also names `CONTEXT.md`.

**Desired behavior:**
The format spec lives at `GLOSSARY-FORMAT.md`, git history preserved across the move, sibling-named with `ADR-FORMAT.md`. Its example block uses `## Terms` instead of `## Language`. The "Single vs multi-context repos" section is gone — no `CONTEXT-MAP.md` example, no skill-inference list, no escape hatch. The skill prompt's "File structure" subsection collapses to a single line of prose describing the layout (a `GLOSSARY.md` at the repo root, plus `docs/adr/` for ADRs); the multi-context ASCII tree is removed. The prompt's link target updates to `GLOSSARY-FORMAT.md`; every remaining `CONTEXT.md` mention in the prompt — body and frontmatter `description:` — becomes `GLOSSARY.md`. The consumer-side `framework/domain.md` opens with a list pointing at `GLOSSARY.md` and describes the file layout in one line of prose; the "This is a single-context repo:" framing and ASCII tree are gone, and the "Use the glossary's vocabulary" section references `GLOSSARY.md`.

**Key interfaces:**
- **Format spec contract** — the H1 becomes `# GLOSSARY.md Format`. The example block's inner H2 is `## Terms` (was `## Language`). Term entry shape (`**Name**:` / body / `_Avoid_:`), Relationships, Example dialogue, Flagged ambiguities, and the Rules list are preserved verbatim. The "Single vs multi-context repos" trailing section is removed in its entirety — no replacement section.
- **Skill prompt contract** — the YAML frontmatter `name:` (`grill-with-docs`) is unchanged. The frontmatter `description:` field has `CONTEXT.md` replaced with `GLOSSARY.md`. In the body, the "File structure" subsection is replaced by one line of prose: enough to tell the reader where the glossary and ADRs live, no ASCII tree. The `[CONTEXT-FORMAT.md](./CONTEXT-FORMAT.md)` link becomes `[GLOSSARY-FORMAT.md](./GLOSSARY-FORMAT.md)`. Any other `CONTEXT.md` mentions in the prompt body (section headings, prose) become `GLOSSARY.md`.
- **Consumer-side domain-doc contract** — `framework/domain.md`'s "Before exploring, read these" list opens with `GLOSSARY.md`. The "File structure" H2 either disappears (replaced by a single inline sentence elsewhere in the doc) or collapses to one prose line; the "single-context repo" framing and the ASCII tree do not survive in any form. The "Use the glossary's vocabulary" section references `GLOSSARY.md`. The `docs/formann/domain.md` symlink resolves through `.formann → framework`, so editing `framework/domain.md` is sufficient — do not edit the symlink target separately.
- **Git move discipline** — `git mv framework/skills/grill-with-docs/CONTEXT-FORMAT.md framework/skills/grill-with-docs/GLOSSARY-FORMAT.md` as a recorded rename. Heading and content edits may be staged together with the move or in a follow-up edit; what matters is that git records a rename for the file.
