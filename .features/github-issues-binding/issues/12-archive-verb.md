# Archive verb (`/triage` archive against the GH binding)

> *This was generated by AI during to-issues.*

## Parent

`.features/github-issues-binding/PRD.md`

## What to build

Document and realize the **Archive** verb for the GH binding. `/triage`'s archive action under this binding closes the parent issue with `state_reason=completed` and applies `formann:archived`.

Mechanics:

- `gh issue close <parent-N> --reason completed`
- `gh issue edit <parent-N> --add-label formann:archived`

Effects:

- `tracker-snapshot --list` filters on `is:open label:formann:feature`, so closed parents drop out naturally ÔÇö archived features don't appear in feature discovery.
- The `formann:archived` label is the durable archive marker visible after reopen (closed-state could be reversed; the label survives reopens as the intentional archive signal).
- Sub-issues remain in whatever state they had at archive time (typically all `done` / `wontfix`).

`/triage`'s SKILL.md stays binding-agnostic; the archive realization lives in BINDING.md.
