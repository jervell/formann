# Runner refactor: `(feature, N)` plumbing and `binding_native_ref` helper

> *This was generated by AI during to-issues.*

## Parent

`.features/github-issues-binding/PRD.md`

## What to build

Refactor the runner so it never regex-parses `<feature>/NN` to derive the feature and never assumes the binding-native ref shape. This is binding-agnostic prep work: it lands before any GH-specific code and the local-markdown path stays identical end-to-end.

Three call sites consume the ref by regex-parsing today; each switches to consuming a `(feature, N)` pair plus, where needed, the binding-native ref form retrieved via a new helper:

1. **`dispatch_one`** — the snapshot's `feature` field and the issue's `nn` are both in scope. Pass them explicitly to downstream helpers instead of regex-parsing the ref. When the binding-native ref needs to be passed into a binding script, resolve it via `binding_native_ref`.
2. **`write_abort_flag`** — `.runner-state/aborted/<feature>/<NN>`. Continues to use `<feature>/<N>` as the filesystem path; both halves come from the `(feature, N)` pair, never from regex-parsing the binding-native ref.
3. **Classifier invocations** — wherever the runner passes the ref into the classifier, it now passes `(feature, N)` and resolves the binding-native ref via the helper if the classifier needs it.

New helper: `binding_native_ref <feature> <N>` looks up the binding-native ref by indexing into the current snapshot's issues array. The runner's CLI argument parser (`--issue <feature>/<N>`, regex `^([a-z0-9][a-z0-9-]*)/([0-9]+)$`) is unchanged — that's the **portable** form for operator ergonomics.

Also in this slice (binding-agnostic, related):

- Move HEAD-archive logic out of the `take_snapshot` wrapper and into `framework/bindings/issue-tracker/local-markdown/tracker-snapshot`. The wrapper shrinks to a thin shell-out; each binding owns its "current state of the world" definition.

**Current behavior:**
Three call sites in `framework/runner/run-the-queue.sh` consume the ref by regex-parsing `^([a-z0-9][a-z0-9-]*)/([0-9]+)$` — the CLI argument parser (`ARG_ISSUE_REF`), `write_abort_flag`, and `dispatch_one`. The CLI parser is the **operator-facing** form; the other two regex-parses are inside helpers where the feature and `N` are already in scope from the snapshot.

The `take_snapshot` wrapper in `run-the-queue.sh` extracts `HEAD`'s `.features/` into a temp dir via `git archive | tar`, then invokes the role-surface `tracker-snapshot` against that temp tree. This "current state of the world" definition (the runner-checkout's committed `.features/`) is local-markdown-specific — under github-issues, the equivalent definition is "the live GitHub state" and lives entirely inside the GH binding's `tracker-snapshot`. The HEAD-archive logic does not belong in the binding-agnostic wrapper.

**Desired behavior:**
A new helper `binding_native_ref <feature> <N>` lives in `framework/runner/run-the-queue.sh` (or a sibling lib file). It looks up the binding-native ref by indexing into the **current snapshot**'s `issues` array — never by string-shape inference. It fails loudly (non-zero with stderr message) when `(feature, N)` isn't in the snapshot.

Each of the three downstream call sites consumes `(feature, N)` instead of a parsed ref:

1. **`dispatch_one`** — the snapshot's `feature` field and the issue's `nn` are both in scope at the call site. Pass them explicitly to downstream helpers. When a binding-native ref needs to flow into a binding script, resolve via `binding_native_ref`.

The runner's CLI argument parser stays unchanged: `--issue <feature>/<N>` with the existing `^([a-z0-9][a-z0-9-]*)/([0-9]+)$` regex. That's the **portable** form for operator ergonomics.

The `take_snapshot` wrapper shrinks to a thin shell-out — it no longer extracts HEAD's `.features/` itself. The HEAD-archive logic moves into `framework/bindings/issue-tracker/local-markdown/tracker-snapshot`: when invoked from the runner, the local-markdown snapshot script defines "current state of the world" as the runner-checkout's HEAD-committed `.features/`. Each binding owns its definition of current state.

**Decisions made:**
