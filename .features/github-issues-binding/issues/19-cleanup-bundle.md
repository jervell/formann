# Cleanup bundle: ref-shape language + binding-script hygiene

> *This was generated by AI during to-issues.*

## Parent

`.features/github-issues-binding/PRD.md`

## What to build

Four small fixes, each tiny in isolation but worth bundling so they ride one review pass:

1. **`framework/runner/review-and-gate.md:7`** — Replace *"It looks like `<feature>/<NN>` (canonical form)."* with binding-neutral language. The runner appends the binding-native ref; under github-issues that's `#42`, not `<feature>/<NN>`. Point at `docs/formann/issue-tracker/BINDING.md` for the per-binding ref shape.

2. **`framework/bindings/issue-tracker/github-issues/tracker-snapshot:220`** — The 0-match path interpolates `$_slug` into JSON via `printf '%s'`, then pipes to `jq -c '.'`. Use the same `$(printf '%s' "$_slug" | jq -Rs '.')` pattern that line 322 already uses for proper JSON encoding. Defence against a slug containing `"` or `\` — unreachable today given the slug grammar, but the two paths should not diverge.

3. **`framework/bindings/issue-tracker/github-issues/body-edit`** — The awk script compares `$0 == heading` for exact match. The local-markdown `tracker-snapshot` explicitly strips `\r` before comparison and ships a CRLF regression test (`tracker-snapshot.bats:205`). `body-edit` doesn't. Strip `\r` from each input line so a CRLF-encoded body's `## Agent Brief\r` heading still matches.

4. **`framework/runner/run-the-queue.sh:1207`** (`collect_binding_env`) — On a malformed-line error, the message echoes the entire offending line: `echo "runner: sandbox-env emitted malformed line (...): $line"`. If a future binding's `sandbox-env` ever emits a typo'd `gH_TOKEN=ghp_…` (uppercase-key regex fails), the token reaches stderr, terminal, and `runner.log`. Echo only the key portion (`${line%%=*}`) or a fixed message, never the value.
