# Make `test_helper` symlink portable for host bats runs

> *This was generated by AI during to-issues.*

## Parent

`.features/github-issues-binding/PRD.md`

## What to build

`framework/bindings/issue-tracker/github-issues/tests/test_helper` is currently an absolute symlink to `/repo/framework/runner/tests/test_helper`. The `/repo` mount only exists inside the AFK runner's dispatch container; on host, all 34 binding bats tests fail at `setup()` with:

```
bats_load_safe: Could not find '…/test_helper/bats-support/load'
```

Replace the absolute symlink with a relative one (`../../../../runner/tests/test_helper`) so host bats runs work the same as in-container runs. The local-markdown tests already get this for free because their `test_helper` is a real directory at `framework/runner/tests/test_helper`.

## Acceptance criteria

- [ ] `framework/bindings/issue-tracker/github-issues/tests/test_helper` is a symlink whose target resolves on host (not rooted at `/repo`).
- [ ] `bats framework/bindings/issue-tracker/github-issues/tests/*.bats` runs all four files to completion on host (no setup-time failures).
- [ ] All previously-passing tests continue to pass after the symlink swap (the test files themselves are not touched).
- [ ] In-container test runs (inside the AFK runner's sandbox) still work.

## Blocked by

None — can start immediately.
