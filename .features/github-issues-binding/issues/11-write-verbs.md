# Write verbs (`/triage` against the GH binding)

> *This was generated by AI during to-issues.*

## Parent

## What to build

Document and realize the five write-shaped verbs that `/triage` (and indirectly other skills) consume. Under the GH binding these map to label add/remove, close-with-reason, `gh issue comment`, and section-level body splices via `body-edit`.

Verbs:

- **Set the state to `<state>`** — non-terminal: `gh issue edit --remove-label formann:status:<prev>` + `gh issue edit --add-label formann:status:<new>`. Terminal: `gh issue close --reason completed` (for `done`) or `gh issue close --reason not_planned` (for `wontfix`); reopening from terminal removes the close-reason.
- **Set issue metadata** — `category` and `type`: label add/remove (`formann:category:<cat>`, `formann:type:<type>`). `blocked-by`: section-level body edit (via `body-edit`) into the `## Blocked by` section. Blocker refs use the binding-native `#N` form so GitHub autolinks them.
- **Comment with `<kind>`** — `gh issue comment <N> --body @content.md`. The comment's first content line is `### <Kind> — <YYYY-MM-DD>` (UTC date). Web-UI comments without this heading are treated as unstructured. AI-disclaimer convention transfers: AI-produced comments lead with `> *This was generated by AI during <skill>.*` on their first content line.
- **Publish the agent brief** — `body-edit <N> "Agent Brief" @brief.md`. Section-shaped write; whole-body in mechanics (see slice 04's documented residual race).
- **Record triage notes** — `body-edit <N> "Triage Notes" @notes.md`. On transition out of `needs-info`, the verb realization is `body-edit <N> "Triage Notes" /dev/null` (deletes the section).

`/triage`'s SKILL.md stays binding-agnostic; the realizations live in BINDING.md.
