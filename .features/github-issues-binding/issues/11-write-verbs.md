# Write verbs (`/triage` against the GH binding)

> *This was generated by AI during to-issues.*

## Parent

`.features/github-issues-binding/PRD.md`

## What to build

Document and realize the five write-shaped verbs that `/triage` (and indirectly other skills) consume. Under the GH binding these map to label add/remove, close-with-reason, `gh issue comment`, and section-level body splices via `body-edit`.

Verbs:

- **Set the state to `<state>`** — non-terminal: `gh issue edit --remove-label formann:status:<prev>` + `gh issue edit --add-label formann:status:<new>`. Terminal: `gh issue close --reason completed` (for `done`) or `gh issue close --reason not_planned` (for `wontfix`); reopening from terminal removes the close-reason.
- **Set issue metadata** — `category` and `type`: label add/remove (`formann:category:<cat>`, `formann:type:<type>`). `blocked-by`: section-level body edit (via `body-edit`) into the `## Blocked by` section. Blocker refs use the binding-native `#N` form so GitHub autolinks them.
- **Comment with `<kind>`** — `gh issue comment <N> --body @content.md`. The comment's first content line is `### <Kind> — <YYYY-MM-DD>` (UTC date). Web-UI comments without this heading are treated as unstructured. AI-disclaimer convention transfers: AI-produced comments lead with `> *This was generated by AI during <skill>.*` on their first content line.
- **Publish the agent brief** — `body-edit <N> "Agent Brief" @brief.md`. Section-shaped write; whole-body in mechanics (see slice 04's documented residual race).
- **Record triage notes** — `body-edit <N> "Triage Notes" @notes.md`. On transition out of `needs-info`, the verb realization is `body-edit <N> "Triage Notes" /dev/null` (deletes the section).

`/triage`'s SKILL.md stays binding-agnostic; the realizations live in BINDING.md.

## Acceptance criteria

- [ ] BINDING.md gains five verb sections: **Set the state**, **Set issue metadata**, **Comment with `<kind>`**, **Publish the agent brief**, **Record triage notes**, each with concrete `gh` / `body-edit` commands.
- [ ] **Set the state**: non-terminal transitions use `--remove-label`/`--add-label`; terminal `done` uses `gh issue close --reason completed`; terminal `wontfix` uses `gh issue close --reason not_planned`. Reopening reverts the close state.
- [ ] **Set the state**: re-running a transition to the current state is a no-op (idempotent label add + label remove of the same label is harmless).
- [ ] **Set issue metadata** (category/type): label-flip semantics — remove the old `formann:category:*` (or `formann:type:*`) and add the new one in a single logical operation.
- [ ] **Set issue metadata** (blocked-by): `## Blocked by` body section is rewritten via `body-edit`. Blocker refs in the rewritten section use `#N` form (GitHub autolinks them).
- [ ] **Comment with `<kind>`**: comment body's first content line is `### <Kind> — <YYYY-MM-DD>` (UTC). The AI-disclaimer (if produced by a skill) appears immediately after the heading on its own line. Web-UI comments without the heading are documented as "unstructured" in BINDING.md.
- [ ] **Comment with `<kind>`**: same-day same-kind collisions get the numeric-suffix convention from local-markdown (`### Implementation (2) — 2026-05-08`), so the parse rule stays portable.
- [ ] **Publish the agent brief**: `body-edit <N> "Agent Brief" @brief.md` replaces or appends the section. Re-running overwrites the prior brief; surrounding sections preserved (per slice 04's tests).
- [ ] **Record triage notes**: `body-edit <N> "Triage Notes" @notes.md` writes the section; on transition out of `needs-info`, the verb passes `/dev/null` (or an empty file) and `body-edit` removes the section.
- [ ] BINDING.md's "Authorship" section documents the weakened semantics: all writes come from the maintainer's PAT, so git-blame attribution is absent — the AI disclaimer is the only authorship signal.
- [ ] BINDING.md notes that multi-call atomicity isn't guaranteed (e.g., comment posted but state-flip rate-limited on retry); the runner's classifier surfaces partial failures and the maintainer reconciles.
- [ ] End-to-end verification on the scratch feature from slice 08/09: `/triage` can transition states, set category/type, comment with kind, publish an agent brief, and record/clear triage notes against a sub-issue.
