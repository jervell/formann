## What to build

Write `docs/adr/0005-github-issues-binding-object-model.md` recording two architectural choices the PRD makes:

1. **Parent issue with sub-issues** as the feature/slice modelling shape, over milestones, labels-only, and GitHub Projects v2.
2. **Slug as a `formann:slug:<slug>` label** on the parent issue, over body-marker, title-prefix, and no-slug-storage.
